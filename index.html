02. El Problema

Fragmentación
Operativa

Las MYPES peruanas representan el 96% del mercado, pero operan sus procesos financieros de forma desconectada.

El problema central radica en que los comprobantes electrónicos (XMLs) se envían solo por cumplimiento legal, perdiendo la oportunidad de usarlos como activos estratégicos para la toma de decisiones.

Punto Ciego Tributario

Incapacidad de verificar si un proveedor es SSCO (Sujeto Sin Capacidad Operativa) antes de la compra.

Ineficiencia de Inventario

Capital inmovilizado por falta de algoritmos de predicción de demanda por SKU.

Propuesta Innovadora (SaaS)

SaaS Inteligente

SENTINEL convierte el flujo de comprobantes electrónicos en el núcleo de un motor de inteligencia artificial. Integramos la facturación SUNAT con motores de detección de fraude y predicción de demanda en una plataforma unificada.

Motor de ML

Ejes del Sistema

  • Integración SUNAT UBL 2.1 unificada.
  • Detección de Fraude SSCO Real-Time.
  • Forecasting de demanda por SKU/categoría.

Arquitectura de Microservicios

Diseño desacoplado y escalable optimizado para el procesamiento masivo de XMLs y modelos de IA.

UI/UX Engine

Desarrollado en Angular 19+ para máxima responsividad y performance.

Backend Core

Spring Boot 3 con arquitectura hexagonal y orquestación Docker.

Persistence Layer

PostgreSQL + TimescaleDB para analítica de series temporales.

ML Pipeline

Integración con Python para modelos LightGBM y SHAP.

Alcance & Límite

Incluye

• Emisión y recepción de facturas UBL 2.1 con envío directo a SUNAT.
• Verificación RUC en tiempo real contra base SSCO (Sujeto Sin Capacidad Operativa).
• Predicción de demanda y generación automática de sugerencias de compra.
• Dashboard interactivo con KPIs financieros y cumplimiento Ley 29733 (Cifrado).

No Incluye

• Módulo de contabilidad completa (Balance General, Libro Diario).
• Gestión de planillas, RRHH o control de asistencia.
• Aplicación móvil nativa iOS/Android.
• Emisión de comprobantes para regímenes especiales (Bancos, Seguros).
